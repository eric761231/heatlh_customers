# 📱 中文輸入說明

## ❓ 問題：模擬器只能輸入英文

這是**模擬器的問題**，不是應用程式的問題。

### 🔍 原因

Android 模擬器預設可能沒有安裝中文輸入法，所以只能輸入英文。

### ✅ 解決方法

#### 方法 1：在模擬器中安裝中文輸入法（推薦）

1. **打開模擬器的設定**
   - 在模擬器中，點擊「設定」圖示
   - 或從通知欄下拉，點擊「設定」

2. **進入語言和輸入法設定**
   - 找到「系統」→「語言和輸入法」
   - 或「System」→「Languages & input」

3. **安裝中文輸入法**
   - 點擊「虛擬鍵盤」或「Virtual keyboard」
   - 點擊「管理鍵盤」或「Manage keyboards」
   - 啟用「Gboard」或「Google 鍵盤」
   - 在 Gboard 設定中，添加「繁體中文」或「簡體中文」

4. **切換輸入法**
   - 在輸入框中長按，選擇「輸入法選項」
   - 或從通知欄切換輸入法

#### 方法 2：使用 Android Studio 的 AVD Manager 設定

1. **打開 AVD Manager**
   - 在 Android Studio 中，點擊「Tools」→「Device Manager」
   - 或「工具」→「裝置管理員」

2. **編輯模擬器**
   - 找到您使用的模擬器
   - 點擊「編輯」圖示（鉛筆圖示）

3. **選擇系統映像**
   - 選擇包含中文輸入法的系統映像
   - 建議使用「Google APIs」版本（不是「Google Play」版本）

#### 方法 3：下載 Gboard 應用程式

1. **在模擬器中打開 Google Play**
2. **搜尋「Gboard」**
3. **安裝 Gboard**
4. **在 Gboard 設定中添加中文輸入法**

## 📦 打包後在真實手機上會怎樣？

### ✅ **不會有這個問題！**

原因：

1. **真實手機通常都有中文輸入法**
   - 台灣/香港/中國的手機預設都有中文輸入法
   - 即使沒有，用戶也可以從 Google Play 下載

2. **Flutter 應用程式完全支援中文輸入**
   - Flutter 的 `TextField` 和 `TextFormField` 預設支援所有輸入法
   - 不需要特別配置
   - 應用程式會自動使用手機上已安裝的輸入法

3. **應用程式配置正確**
   - 您的 `AndroidManifest.xml` 已經正確配置
   - `android:windowSoftInputMode="adjustResize"` 確保輸入法不會遮擋輸入框

## 🔧 應用程式配置檢查

### ✅ 已正確配置的項目

1. **AndroidManifest.xml**
   ```xml
   android:windowSoftInputMode="adjustResize"
   ```
   這確保輸入法出現時，應用程式會自動調整大小。

2. **TextFormField 配置**
   - 所有文字輸入欄位都使用 `TextFormField`
   - 沒有限制輸入類型（除了 email、phone、number 等特定欄位）
   - 預設支援所有輸入法

### 📝 輸入欄位類型說明

| 欄位類型 | 使用的 keyboardType | 是否支援中文 |
|---------|-------------------|------------|
| 姓名、地址等 | 預設（無 keyboardType） | ✅ 完全支援 |
| Email | `TextInputType.emailAddress` | ✅ 支援（可切換輸入法） |
| 電話 | `TextInputType.phone` | ✅ 支援（可切換輸入法） |
| 數字 | `TextInputType.number` | ⚠️ 可能只顯示數字鍵盤（但可切換） |

## 💡 開發建議

### 在模擬器上測試中文輸入

1. **安裝中文輸入法**（使用上面的方法 1）
2. **測試所有文字輸入欄位**
3. **確認中文輸入正常**

### 在真實手機上測試

1. **打包應用程式**
   ```bash
   cd flutter_app
   flutter build apk
   ```

2. **安裝到手機**
   - 將生成的 APK 傳輸到手機
   - 安裝並測試

3. **確認中文輸入正常**
   - 測試所有輸入欄位
   - 確認中文輸入、選字都正常
