# 🔥 熱更新快速指南

## ⚡ 快速操作

### 當應用程式正在運行時：

| 操作 | 按鍵 | 用途 | 速度 |
|------|------|------|------|
| **熱重載** | `r` | 快速更新 UI 和簡單邏輯 | ⚡ 最快 |
| **熱重啟** | `R` | 重新啟動應用程式（保留狀態） | ⚡ 快 |
| **停止應用** | `q` | 完全停止應用程式 | - |

## 📝 使用場景

### 使用 `r` (熱重載) 當您修改：
- ✅ UI 文字內容
- ✅ 顏色值
- ✅ 簡單的 Widget 屬性
- ✅ 樣式調整
- ✅ 簡單的邏輯修改

### 使用 `R` (熱重啟) 當您修改：
- ✅ 主題配置 (`ThemeData`)
- ✅ 顏色配置 (`AppColors`)
- ✅ 路由配置
- ✅ 初始化邏輯
- ✅ 狀態管理配置

### 需要完全重啟 (`q` + `flutter run`) 當您：
- ✅ 新增或修改依賴 (`pubspec.yaml`)
- ✅ 修改 `main.dart` 的初始化邏輯
- ✅ 修改資料模型結構
- ✅ 遇到編譯錯誤

## 🚀 標準工作流程

### 1. 啟動應用程式
```bash
cd flutter_app
flutter run
```

### 2. 修改代碼後

**選項 A：快速更新（推薦）**
```
在終端按：r
```

**選項 B：如果 `r` 沒有效果**
```
在終端按：R
```

**選項 C：如果還是看不到變化**
```
在終端按：q
然後重新執行：flutter run
```

## 🔧 常見問題解決

### ❌ 問題：按 `r` 沒有反應

**解決步驟：**
1. 確認終端視窗有焦點（點擊終端）
2. 確認應用程式正在運行（看到 "Flutter run key commands" 訊息）
3. 嘗試按 `R` 進行熱重啟
4. 如果還是不行，按 `q` 退出後重新運行

### ❌ 問題：修改顏色後看不到變化

**解決步驟：**
1. 確認已保存檔案（Ctrl+S）
2. 按 `R` 進行熱重啟（顏色配置需要重啟）
3. 如果還是不行：
   ```bash
   # 停止應用（按 q）
   flutter clean
   flutter pub get
   flutter run
   ```

### ❌ 問題：修改主題後還是舊樣式

**解決步驟：**
1. 按 `R` 進行熱重啟（主題變更需要重啟）
2. 確認 `main.dart` 中的 `ThemeData` 已更新
3. 確認已導入 `app_colors.dart`

### ❌ 問題：編譯錯誤

**解決步驟：**
1. 查看終端錯誤訊息
2. 修正代碼錯誤
3. 保存檔案
4. 按 `r` 或 `R` 重新載入

## 💡 最佳實踐

### ✅ 開發時建議

1. **保持應用程式運行**
   - 不要頻繁停止和重啟
   - 使用 `r` 和 `R` 進行快速迭代

2. **保存檔案**
   - 修改後記得保存（Ctrl+S）
   - Flutter 會自動檢測檔案變更

3. **查看終端輸出**
   - 注意 "Reloaded" 或 "Restarted" 訊息
   - 如果有錯誤，會顯示在終端

4. **測試順序**
   - 先試 `r`（最快）
   - 不行再試 `R`（熱重啟）
   - 最後才完全重啟

## 🎯 快速檢查清單

修改代碼後，按順序檢查：

- [ ] 檔案已保存（Ctrl+S）
- [ ] 終端有焦點（點擊終端視窗）
- [ ] 按 `r` 進行熱重載
- [ ] 如果沒效果，按 `R` 進行熱重啟
- [ ] 查看終端是否有錯誤訊息
- [ ] 如果還是不行，按 `q` 退出後重新運行

## 📱 在模擬器上測試

### 確認應用程式正在運行
終端應該顯示：
```
Flutter run key commands.
r Hot reload. 🔥🔥🔥
R Hot restart.
q Quit (terminate the application on the device).
```

### 修改代碼後
1. 保存檔案（Ctrl+S）
2. 在終端按 `r`
3. 等待 "Reloaded" 訊息
4. 在模擬器中查看變化

## 🆘 如果還是不行

執行完整清理：

```bash
cd flutter_app

# 1. 停止應用（如果正在運行，按 q）
# 2. 清理快取
flutter clean

# 3. 重新獲取依賴
flutter pub get

# 4. 重新運行
flutter run
```
